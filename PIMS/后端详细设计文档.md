# 物业管理系统（PIMS）后端详细设计文档

## 1. 系统概述

### 1.1 技术选型
- **编程语言**: Java 8 (JDK 1.8)
- **开发框架**: Spring Boot 2.7.x
- **数据库**: MySQL 8.0
- **ORM框架**: MyBatis-Plus
- **安全框架**: Spring Security + JWT
- **工具类**: Hutool、Lombok
- **API文档**: Knife4j (Swagger增强版)

## 3. 核心业务逻辑设计

### 3.1 用户注册流程
1. **输入验证**
   - 验证手机号格式（11位，符合中国大陆手机号规范）
   - 验证房号格式（Xxxx格式，如A310）
   - 检查房号唯一性（房号不能重复）

2. **验证码生成**
   - 生成6位随机数字验证码
   - 打印到后台日志（模拟短信发送）
   - 将验证码存储到Redis（5分钟过期）

3. **注册逻辑**
   - 验证验证码正确性
   - 密码加密（BCrypt）
   - 创建用户记录（status=0，未入住）
   - 返回注册成功

### 3.2 用户登录流程
1. **支持两种登录方式**
   - 方式1：房号 + 密码
   - 方式2：手机号 + 密码

2. **登录验证**
   - 查询用户是否存在
   - 验证密码是否正确
   - 检查用户状态（已注销用户不能登录）
   
3. **Token生成**
   - 生成JWT Token（有效期24小时）
   - 返回用户信息和Token

### 3.3 找回密码流程
1. 输入注册时的手机号
2. 生成验证码并打印到日志
3. 验证验证码
4. 更新密码（加密存储）

### 3.4 管理员登录流程
1. 验证用户名和密码
2. 检查是否为超级管理员
3. 生成JWT Token
4. 返回管理员信息和Token

### 3.5 入住申请与审核流程
1. **用户发起入住申请**
   - 填写姓名、身份证号、住房面积
   - 验证身份证号格式（18位）
   - 创建审核记录（status=0，待审核）

2. **管理员审核**
   - 查看待审核列表
   - 选择同意/拒绝
   - **同意**：
     - 更新审核记录状态为1
     - 创建住户信息记录
     - 更新用户入住状态为1
     - 记录注册时间（用于计算缴费日）
   - **拒绝**：
     - 更新审核记录状态为2
     - 用户需重新申请

### 3.6 住户信息管理
1. **查询功能**
   - 支持多字段条件查询（房号、姓名、手机号等）
   - 支持排序（升序/降序）
   - 分页展示

2. **修改功能**
   - 管理员可修改物业费单价
   - 管理员可修改入住状态

3. **退房逻辑**
   - 检查是否有未缴清的物业费
   - 未缴清不允许退房
   - 缴清后设置入住状态为0
   - 逻辑删除用户和住户信息

### 3.7 缴费管理
1. **立即缴费**
   - 计算金额：面积 × 物业费单价
   - 创建缴费记录
   - 记录缴费月份
   - 模拟支付成功

2. **缴费记录查询**
   - 按日期区间查询
   - 计算区间内缴费总额
   - 分页展示

### 3.8 缴费提醒定时任务
1. **定时任务配置**
   - 每天凌晨1点执行
   - 扫描所有已入住用户

2. **提醒规则**
   - 计算每个用户的缴费日：注册时间的异月同日
   - 提前5天发送提醒
   - 检查当月是否已缴费
   - 未缴费则发送消息提醒

3. **消息内容**
   - 标题：物业费缴费提醒
   - 内容："尊敬的住户，您本月即将缴纳物业费，金额为xxx元，请及时缴费，如已完成缴费请忽略"

### 3.9 管理员管理
1. **超级管理员（admin）权限**
   - 创建新管理员
   - 查看所有管理员
   - 修改所有管理员密码

2. **普通管理员权限**
   - 仅能修改自己的密码
   - 拥有住户信息管理全部权限
   - 拥有入住审核权限

## 4. 安全设计

### 4.1 密码安全
- 使用BCrypt加密算法
- 密码不明文存储
- 密码强度建议（前端校验）

### 4.2 接口安全
- JWT Token认证
- Token有效期24小时
- 刷新Token机制
- 权限校验（管理员/普通用户）

### 4.3 数据验证
- 手机号格式验证（正则）
- 身份证号格式验证（正则 + 校验位）
- 房号格式验证（正则）
- SQL注入防护（MyBatis预编译）
- XSS防护（输入过滤）

### 4.4 验证码安全
- 验证码有效期5分钟
- 验证码使用后立即失效
- 同一手机号60秒内只能发送一次

## 5. 接口设计原则

### 5.1 统一返回格式
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 5.2 统一错误码
- 200: 成功
- 400: 参数错误
- 401: 未认证
- 403: 无权限
- 404: 资源不存在
- 500: 服务器错误
- 1001: 验证码错误
- 1002: 用户名或密码错误
- 1003: 房号已存在
- 1004: 手机号已注册
- 1005: 审核中不能重复申请
- 1006: 物业费未缴清，不能退房

### 5.3 分页格式
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 100,
    "pageNum": 1,
    "pageSize": 10,
    "pages": 10,
    "list": []
  }
}
```

- ERROR: 系统错误、异常信息
- WARN: 业务警告（如登录失败、验证码错误）

- DEBUG: 调试信息

### 6.2 关键日志记录
- 用户注册/登录/登出
- 验证码生成和验证
- 入住申请和审核

- 定时任务执行
